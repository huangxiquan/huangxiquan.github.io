RxJava初体验

发表于 2016-08-11   |   阅读次数   |  

前言

最近学习了一下RxJava,在网上找各种资料,最后发现了一篇收集整理RxJava资料的文章.大家可以去看看,最后建议大家看看官方文档.之所以写这篇文章并不是说我写的要比别人好,而是自己写出来首先印象更加深刻,其次方便以后查找.就当做读后感吧.

RxJava简介

Rx来源于ReactiveX这个组织.这个组织专做各种语言的响应式编程库.专注解决异步编程问题.所以RxJava是针对Java中异步编程的一个解决方案.特点是使得代码相当简洁.写RxJava代码时让我感觉到了python的味道.写起来真的是行云流水,一气呵成.但是入门比较难.它像是一种编程思想,我们很难转变我们的思想.

观察者模式

RxJava用到了一个很重要的模式-观察者模式.如果已经了解观察者模式的人可以跳过本节.
观察者模式中有两个重要的对象.观察者(observer)和被观察的事物(observable).observable类中持有一个列表保存有所有observer对象,当被观察事物发生改变时,observable会去遍历这个列表,并且依次调用observer对象里的update方法.observer类必须实现update方法.
观察者模式的好处在于,观察者不用阻塞等待某件事情的完成.而是当被观察者做完某些事后,会通知观察者.所以这个设计模式非常适合异步操作.

RxJava的Hello World

学一门语言第一件事就是在控制台打印一句Hello World.这似乎已经成了一个习惯.不管你们有没有,反正我有.
由于RxJava是基于观察者设计模式,所以也有观察者(observer或subsriber)和被观察者(Observable).
我们首先创建一个Observable.


Observable observable = Observable.create(new Observable.OnSubscribe(){
            @Override
            public void call(Subscriber<? super String> subscriber) {
                subscriber.onNext(“Hello World”);
                subscriber.onCompleted();
            }
        });

接下来我们创建一个观察者.


Subscriber subscriber = new Subscriber() {
            @Override
            public void onCompleted() {
                Log.i(“hxq”,”onCompleted”);
            }
            @Override
            public void onError(Throwable e) {
                Log.i(“hxq”,”onError”);
            }
            @Override
            public void onNext(Object o) {
                Log.i(“hxq”,”onNext:” + o.toString());
            }
        };

最后将两者关联起来-订阅

observable.subscribe(subscriber);

当observable被订阅时会调用OnSubscribe中的call方法.在call方法里面操作完后调用onNext方法.通知subscriber.

RxJava中的操作符

有的人可能说了:”这代码看着也不简洁啊,和我们做异步操作一样,设置一个监听,然后回调”.那接下来我们就来简化代码.这里要引入RxJava操作符的概念,所谓的操作符也就是方法.叫操作符更贴切.RxJava中的操作符很多,这也正是RxJava强大的地方.我们甚至可以自定义操作符(不过官方不推荐这么做,很容易出现问题).这里我们会用到just操作符.
just的作用是将任何对象转化为Observable,所以创建observable的过程可以简化为

Observable obserable = Observable.just(“Hello World”);

如果我们只在onNext里面做操作而不关心其他两个方法,我们可以在订阅时传递Action1对象,所以最终代码可以简化为:


    Observable.just(“Hello World”)
                .subscribe(new Action1() {
                    @Override
                    public void call(String s) {
                        Log.i(“hxq”,s);
                    }
                });

大家此时可能会有疑惑,subcribe方法不是应该传观察者对象吗?Action1又是个什么鬼?当遇到疑问时,只需要翻翻源码就知道了.这是一个重载方法.只是做了一个简单封装.


public final Subscription subscribe(final Action1<? super T> onNext) {
        if (onNext == null) {
            throw new IllegalArgumentException(“onNext can not be null”);
        }
        return subscribe(new Subscriber() {
            @Override
            public final void onCompleted() {
                // do nothing
            }
            @Override
            public final void onError(Throwable e) {
                throw new OnErrorNotImplementedException(e);
            }
            @Override
            public final void onNext(T args) {
                onNext.call(args);
            }
        });
    }

现在是不是很简洁了.还可以更简洁,那就是引入Lamda表达式,jdk1.8已经支持Lamda表达式了.在android studio上还需要一个gradle插件-retrolambda.最终代码可以简化为:

Observable.just(“Hello World”)
        .subscribe(s -> Log.i(“hxq”,s));

这样的代码看着不得不说很舒服啊.可能有人对lamda表达式不是很理解,这里简单的解释下,->前面的是参数,后面是方法体.是函数的另一种写法.适合写内部类.
接下来在为大家介绍一个比较重要的操作符-map
map的作用是做数据转换,比如服务器给客户端的图片可能只是一个url连接,所以当我们请求到了数据后并不是我们最终需要的,我们可能还需要对他做操作,这时候就需要map.这里举一个简单的例子,将String类型转换为Integer类型.

Observable.just(“3”)
                .map(s -> Integer.parseInt(s))
                .subscribe(s -> Log.i(“hxq”,”Class:” + s.getClass()));

RxJava中还有很多好用的操作符,这里就不一一介绍了.

RxJava中的线程切换

RxJava中做线程切换主要靠Scheduler(调度器).Observable中的subScribeOn()和observeOn()分别用来决定Observable操作执行的线程和Subscriber操作执行的线程.


Observable.just(“3”)
                .map(s -> Integer.parseInt(s))
                .subscribeOn(Schedulers.io())
                .observeOn(AndroidSchedulers.mainThread())
                .subscribe(s -> Log.i(“hxq”,”Class:” + s.getClass()));

代码中,数据转换的过程发生在io线程,输出内容发生在android主线程.轻松切换线程.
